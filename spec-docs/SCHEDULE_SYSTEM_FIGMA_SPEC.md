# Schedule System - Figma Design Specification

> **Feature Area**: Regular Season - Schedule Management
> **Platform**: iPad-first (1024Ã—768 minimum)
> **Created**: January 29, 2026
> **Integration**: Works with Finalize & Advance phase and Today's Game tab

---

## Overview

The Schedule System manages the league-wide game schedule for each season. Unlike traditional schedule generators, this system supports **game-by-game input** since the actual schedule comes from Super Mega Baseball 4 (SMB4).

### Key Principles

1. **SMB4 is the source of truth** - Schedule comes from the game, not generated by the tracker
2. **Flexible input** - Users can pre-build schedule OR add games on-the-fly
3. **League-wide tracking** - All teams' games in one schedule, filterable by team
4. **Auto-pull to Today's Game** - Next unplayed game auto-populates the game tracker
5. **Empty start** - New seasons begin with no scheduled games

### Validation Rules
- Away team cannot equal Home team
- Game number must be unique and positive
- Auto-increment finds highest existing game number + 1

### Series Add Behavior
When "Add as Series" is clicked with count = 3:
- Creates 3 games with same Away/Home teams
- Game numbers: 21, 22, 23 (sequential)
- Days: 21, 22, 23 (sequential)
- All status = SCHEDULED

## Data Model

```typescript
interface ScheduledGame {
  id: string;
  seasonNumber: number;
  gameNumber: number;           // 1, 2, 3... (league-wide sequence)
  dayNumber: number;            // For display: Day 1, Day 2...
  date?: string;                // Optional: "July 12"
  time?: string;                // Optional: "7:00 PM"
  awayTeamId: string;
  homeTeamId: string;
  status: 'SCHEDULED' | 'IN_PROGRESS' | 'COMPLETED' | 'SKIPPED';
  result?: {
    awayScore: number;
    homeScore: number;
    winningTeamId: string;
    losingTeamId: string;
  };
  gameLogId?: string;           // Links to full game data after completion
  createdAt: Date;
  completedAt?: Date;
}
```

## Schedule Logic

### Auto-Pull to Today's Game

```typescript
function getNextScheduledGame(teamFilter?: string): ScheduledGame | null {
  const games = getAllGames()
    .filter(g => g.status === 'SCHEDULED')
    .filter(g => !teamFilter || g.awayTeamId === teamFilter || g.homeTeamId === teamFilter)
    .sort((a, b) => a.gameNumber - b.gameNumber);

  return games[0] || null;
}
```

### After Game Completion

```typescript
function onGameComplete(gameId: string, result: GameResult): void {
  // 1. Mark game as completed
  updateGame(gameId, {
    status: 'COMPLETED',
    result,
    completedAt: new Date()
  });

  // 2. Auto-pull next game
  const nextGame = getNextScheduledGame();

  if (nextGame) {
    // Today's Game tab shows next game
    setCurrentGame(nextGame);
  } else {
    // Show "No games in queue" prompt
    showAddGamePrompt();
  }
}
```

### Game Number Auto-Increment

```typescript
function getNextGameNumber(): number {
  const maxGameNumber = Math.max(
    0,
    ...getAllGames().map(g => g.gameNumber)
  );
  return maxGameNumber + 1;
}
```

### Filter Logic

```typescript
function getFilteredSchedule(filter: 'FULL_LEAGUE' | string): ScheduledGame[] {
  const games = getAllGames();

  if (filter === 'FULL_LEAGUE') {
    return games.sort((a, b) => a.gameNumber - b.gameNumber);
  }

  // Filter by team (show games where team is home OR away)
  return games
    .filter(g => g.awayTeamId === filter || g.homeTeamId === filter)
    .sort((a, b) => a.gameNumber - b.gameNumber);
}
```

## User Flows

### Flow 1: New Season Start
1. User advances to new season via Finalize & Advance
2. Schedule is empty
3. User goes to Schedule tab
4. Sees empty state with [+ Add Game] prompt
5. Adds first game (or series)
6. Today's Game tab now shows that game

### Flow 2: Playing Through Games
1. User has games in schedule
2. Goes to Today's Game tab
3. Sees next scheduled game auto-populated
4. Clicks "Play Game" â†’ Game Tracker
5. Completes game
6. Returns to Today's Game tab â†’ Next game auto-pulled
7. Repeat until queue empty

### Flow 3: Queue Empty Mid-Season
1. User completes last scheduled game
2. Today's Game tab shows "No games in queue"
3. User clicks [Go to Schedule] or [+ Add Game Now]
4. Adds more games
5. Returns to Today's Game â†’ Next game ready

### Flow 4: Pre-Building Schedule
1. User wants to enter full schedule upfront
2. Goes to Schedule tab
3. Uses [+ Add Game] repeatedly OR [+ Add Series]
4. Builds out 48 games (or whatever season length)
5. Today's Game tab auto-pulls from queue as games are played
