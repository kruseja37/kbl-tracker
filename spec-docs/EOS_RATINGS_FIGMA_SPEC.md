## Overview

End-of-Season (EOS) Ratings Adjustments is an **offseason ceremony** where players' ratings change based on their season performance vs salary expectations. This creates a "player development" narrative moment.

### Design Goals
1. **Reveal Drama** - Make rating changes feel like a reveal/announcement
2. **Clear Causation** - Show WHY ratings changed (WAR vs salary percentile)
3. **Manageable Flow** - Don't overwhelm with 200+ players at once
4. **Manager Agency** - Allow managers to distribute their bonus points

### Confirmed Design Decisions
| Decision | Confirmed Approach |
|----------|-------------------|
| **Adjustment Systems** | âœ… BOTH apply: System A (rating adjustments) runs first, then System B (salary adjustments) |
| **Manager Distribution** | âœ… USER-CONTROLLED - users manually allocate points |
| **Reveal Mechanism** | âœ… TEAM-BY-TEAM dramatic reveal with navigation |

## Animation Specifications

### Rating Change Reveal
- **Option A (Subtle)**: Numbers fade from old to new (300ms)
- **Option B (Dramatic)**: Slot machine roll-up effect (800ms)

### Card Expand/Collapse
- Duration: 250ms
- Easing: ease-out
- Height animates smoothly

### Progress Indicators
- Circular progress for remaining points
- Animate on allocation add/remove

### Transition Between Teams
- Slide left/right (300ms)
- Fade overlap

---

## Accessibility

1. **Color Contrast**: All text meets WCAG AA
2. **Change indicators**: Use both color AND icon (not color alone)
3. **Screen reader**: Announce changes as "Power increased from 85 to 88, plus 3"
4. **Touch targets**: 44pt minimum
5. **Reduced motion**: Option to disable animations
